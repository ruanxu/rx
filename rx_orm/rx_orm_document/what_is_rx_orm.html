rx_orm是我在常年开发的过程中积累的一种经验吧！

我从事的是.net开发，在开发的过程中少不了使用orm框架

比如微软自己的ef框架，这个框架我觉得已经很成熟了，但是再好orm都有一个问题？

orm都是真的一种语言或者或者一组关系的，所以又出现一个问题

现在web开发领域的东西南北都有，后台的orm只是解决了后端数据交互问题，前端还是要重新实现一遍，或者编写web api

这一点在现在的h5开发中体现的非常明显，后端的orm解决不了大问题

所以这个时候我有一个想法，那就是前后端模块化的orm，将orm搬入javascript中

这个提议一开始有人强力否定，因为确实是这样，在js中使用orm有安全性问题，但是现阶段解决了明显的不安全问题，后期会继续完善

使用前后端模块化的orm可以大幅度降低前端逻辑数据异步请求的工作量

一部分确定安全性问题非常高的功能可以使用后端orm完成，因为前端与后端orm的代码编写方式90%以上是相同的

所以基本上你会用后端orm就会使用前端orm，反之同理

现在后端orm是用C#语言实现的，以后会扩展出更多的语言的支持

我的资料完全开源，没有为什么，开源了才有更多的人知道和参与我的项目进行改进，谢谢大家，我会继续更新这套项目
